# 本地部署

## 1. 前置依赖

| 组件 | 版本要求 | 说明 |
|------|---------|------|
| Python | 3.10+ | 运行环境 |
| uv | 最新版 | 包管理工具，替代 pip |
| MySQL | 8.x | 主数据库（建议直接复用金毛项目的本地 MySQL，库名 `golden_vs_persian`）|

## 2. 安装 uv

```bash
# Windows (PowerShell)
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"

# macOS / Linux
curl -LsSf https://astral.sh/uv/install.sh | sh
```

## 3. 安装项目依赖

```bash
cd server
uv sync
```

> `uv sync` 会根据 `pyproject.toml` 自动创建虚拟环境并安装所有依赖。

## 4. 配置环境变量

```bash
# 复制配置模板
cp .env.example .env
```

编辑 `.env` 文件，按实际情况修改：

```env
# MySQL（与金毛项目共用一个实例与库，本项目表统一用 hd_ 前缀）
MYSQL_HOST=localhost
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=你的MySQL密码
MYSQL_DATABASE=golden_vs_persian

# 微信小程序（本地调试可留空，但 wx.login 会报 appid 缺失；正式微信登录需填）
WX_APP_ID=
WX_APP_SECRET=

# JWT
JWT_SECRET=please-change-me-in-production
JWT_EXPIRE_HOURS=168
```

## 5. 创建/复用 MySQL 库并初始化表

本项目不独立建库，直接使用金毛项目的 `golden_vs_persian`。确保该库已存在后执行：

```bash
mysql -h localhost -u root -p golden_vs_persian < server/sql/schema.sql
```

> 首次启动服务时也会通过 SQLAlchemy 自动建表（`hd_users`、`hd_records`），与金毛项目的 `users`/`levels`/`rankings` 等表同库共存、互不干扰。

## 6. 启动后端服务

```bash
cd server
uv run uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

启动成功后可以看到：

```
INFO:     Uvicorn running on http://0.0.0.0:8000
INFO:     Application startup complete.
```

## 7. 验证服务

| 地址 | 说明 |
|------|------|
| http://127.0.0.1:8000/docs | Swagger API 文档（自动生成） |
| http://127.0.0.1:8000/redoc | ReDoc API 文档 |

快速测试服务：

```bash
curl http://127.0.0.1:8000/health
```

返回示例：

```json
{"status": "ok"}
```

> 登录接口 `/api/auth/login` 需传入有效的微信 code，建议通过微信开发者工具运行前端联调测试（详见下一步）。

## 8. 配合微信开发者工具调试

1. 微信开发者工具打开 `client` 目录
2. 「详情」→「本地设置」→ 勾选 **「不校验合法域名、web-view(业务域名)、TLS版本以及HTTPS证书」**
3. 编译运行即可连接本地后端

> 本地调试需在 `.env` 中填入真实的 `WX_APP_ID` / `WX_APP_SECRET`，否则 `wx.login()` 拿到的 code 在后端会报 `appid missing`。

## 9. 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| MySQL 连接失败 | 密码错误、库名不存在或服务未启动 | 检查 `.env` 的 MYSQL_* 配置，确认 `golden_vs_persian` 已创建 |
| 微信登录报 appid missing | `.env` 未填 WX_APP_ID/WX_APP_SECRET | 填入正确的微信小程序凭证（微信公众平台获取）后重启服务 |
| 前端请求被拦截 | 域名校验未关闭 | 微信开发者工具勾选「不校验合法域名」|
| 端口 8000 被占用 | 其他进程占用 | 换端口：`--port 8001`，同步修改 `client/global.js` 中的 `serverUrl` |
