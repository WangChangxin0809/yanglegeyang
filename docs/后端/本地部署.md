# 本地部署

## 1. 前置依赖

| 组件 | 版本要求 | 说明 |
|------|---------|------|
| Python | 3.10+ | 运行环境 |
| uv | 最新版 | 包管理工具，替代 pip |
| PostgreSQL | 15+ | 主数据库 |
| Redis | 7+ | 缓存（可选，不可用时自动降级为直查数据库） |
| Docker Desktop | 最新版 | 用于运行 Redis 容器（推荐方式） |

## 2. 安装 uv

```bash
# Windows (PowerShell)
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"

# macOS / Linux
curl -LsSf https://astral.sh/uv/install.sh | sh
```

## 3. 安装项目依赖

```bash
cd server
uv sync
```

> `uv sync` 会根据 `pyproject.toml` 自动创建虚拟环境并安装所有依赖。

## 4. 配置环境变量

```bash
# 复制配置模板
cp .env.example .env
```

编辑 `.env` 文件，按实际情况修改：

```env
# PostgreSQL
PG_HOST=localhost
PG_PORT=5432
PG_USER=postgres
PG_PASSWORD=你的PG密码
PG_DATABASE=yanglegeyang

# Redis
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_DB=0
REDIS_PASSWORD=

# 微信小程序（本地开发可留空，登录会自动降级为 dev-login）
WX_APP_ID=
WX_APP_SECRET=

# JWT
JWT_SECRET=please-change-me-in-production
JWT_EXPIRE_HOURS=168
```

## 5. 创建 PostgreSQL 数据库

```bash
# Windows（需使用 psql 完整路径，或将其加入 PATH）
& "C:\Program Files\PostgreSQL\16\bin\psql.exe" -U postgres -c "CREATE DATABASE yanglegeyang;"

# macOS / Linux
psql -U postgres -c "CREATE DATABASE yanglegeyang;"
```

> 首次启动服务时会自动建表（users、records），无需手动迁移。

## 6. 启动 Redis（Docker 方式）

```bash
# 启动 Docker Desktop 后执行
docker run -d --name redis -p 6379:6379 --restart unless-stopped redis:alpine
```

> Redis 为可选组件。未启动时后端自动降级：缓存操作静默跳过，所有查询直接走数据库。

## 7. 启动后端服务

```bash
cd server
uv run uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

启动成功后可以看到：

```
INFO:     Uvicorn running on http://0.0.0.0:8000
INFO:     Application startup complete.
```

## 8. 验证服务

| 地址 | 说明 |
|------|------|
| http://127.0.0.1:8000/docs | Swagger API 文档（自动生成） |
| http://127.0.0.1:8000/redoc | ReDoc API 文档 |

快速测试登录接口：


## 9. 配合微信开发者工具调试

1. 微信开发者工具打开 `client` 目录
2. 「详情」→「本地设置」→ 勾选 **「不校验合法域名、web-view(业务域名)、TLS版本以及HTTPS证书」**
3. 编译运行即可连接本地后端

> 本地开发时前端会自动降级为 `dev-login`（跳过微信验证），无需配置 AppID/AppSecret。

## 10. 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| PG 连接失败 | 密码错误或服务未启动 | 检查 `.env` 中 `PG_PASSWORD`，确认 PG 服务已运行 |
| Redis 连接失败 | Docker 未启动或容器未运行 | 启动 Docker Desktop，`docker start redis` |
| 微信登录报 appid missing | 未配置 AppID | 正常现象，会自动降级为 dev-login |
| 前端请求被拦截 | 域名校验未关闭 | 微信开发者工具勾选「不校验合法域名」 |
| 端口 8000 被占用 | 其他进程占用 | 换端口：`--port 8001`，同步修改 `client/global.js` 中的 `serverUrl` |
