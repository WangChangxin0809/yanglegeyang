# 后端总览

## 功能范围

后端只负责两件事：

1. **用户管理** — 存储用户基本信息（微信登录后自动创建）
2. **通关记录** — 存储每关的通关时间

## 技术选型

- **语言**：Python 3.10+
- **框架**：FastAPI
- **异步支持**：uvicorn（ASGI 服务器）
- **包管理**：uv
- **ORM**：SQLAlchemy 2.0（异步模式）
- **数据库**：PostgreSQL + asyncpg
- **缓存**：Redis
- **鉴权**：微信 OpenID + JWT Token

## API 接口设计

### 1. 登录鉴权

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/auth/login` | 微信登录（code 换 token） |

### 2. 通关记录

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/record/submit` | 提交通关记录（关卡ID + 通关时间） |
| GET  | `/api/record/list`   | 获取当前用户的通关记录 |

## 核心流程

### 登录流程

```
小游戏端 wx.login() 获取 code
       ↓
POST /api/auth/login { code }
       ↓
后端用 code 调用微信接口换取 openid + session_key
       ↓
查找/创建用户，生成 JWT Token 返回
       ↓
小游戏端存储 Token，后续请求携带 Authorization 头
```

### 通关记录流程

```
玩家通关 → 前端计算通关时间 → POST /api/record/submit → 后端存入 PostgreSQL
```

## 缓存策略

| 缓存项 | Key 格式 | 过期时间 | 说明 |
|--------|---------|---------|------|
| 用户会话 | `session:{openid}` | 7 天 | 存储 session_key |
| 通关记录 | `record:{user_id}` | 10 分钟 | 缓存用户通关列表，避免重复查库 |
