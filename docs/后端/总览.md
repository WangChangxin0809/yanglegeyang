# 后端总览

## 功能范围

后端只负责两件事：

1. **用户管理** — 存储用户基本信息（微信登录后自动创建）
2. **通关记录** — 存储每关的通关时间

## 技术选型

- **语言**：Python 3.13+
- **框架**：FastAPI
- **异步支持**：uvicorn（ASGI 服务器）
- **包管理**：uv
- **ORM**：SQLAlchemy 2.0（异步模式）
- **数据库**：MySQL 8.x + asyncmy（与「金毛大战波斯猫」项目共用一个 MySQL 实例，本项目表统一用 `hd_` 前缀）
- **连接池健康**：`pool_pre_ping=True` + `pool_recycle=1800`，避免云部署环境下 TCP 空闲超时导致的死连接
- **鉴权**：微信 OpenID + JWT Token

## API 接口设计

### 1. 登录鉴权

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/auth/login` | 微信登录（code 换 token，静默） |
| POST | `/api/auth/update-profile` | 微信授权后上传昵称/头像（需 token） |

### 2. 通关记录

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/record/submit` | 提交通关记录（关卡ID + 通关时间） |
| GET  | `/api/record/list`   | 获取当前用户的通关记录 |

## 核心流程

### 登录流程

授权作为进入游戏的前置场景。loading 完成后：
- 本地已有 token 且有昵称头像 → 直接进菜单（menu）
- 否则 → 强制进入授权场景（auth），用户必须点击微信授权按钮才能继续

```
小游戏端 loading → 检查本地 token
       ↓ 无
进入 auth 场景弹出'微信授权登录'按钮（wx.createUserInfoButton）
       ↓ 点击后
同时拿到 code（wx.login）+ userInfo（昵称/头像）
       ↓
POST /api/auth/login { code } → 换回 JWT token
       ↓
POST /api/auth/update-profile { nickname, avatar_url } → 写回后端
       ↓
token + userInfo 持久化到 wx.setStorageSync，切换到 menu 场景
```

## 通关记录流程

```
玩家通关 → 前端计算通关时间（Date.now() - levelStartTime）→ POST /api/record/submit → 后端存入 MySQL（`hd_records` 表）
```

## 连接池配置

由于云部署环境下反向代理 / 云防火墙 / MySQL `wait_timeout` 会静默切断空闲 TCP 连接，导致长时间空闲后第一次请求报 `TCPTransport closed=True`。解决方案见 [server/app/core/database.py](../../server/app/core/database.py)：

```python
engine = create_async_engine(
    settings.DATABASE_URL,
    echo=settings.DEBUG,
    pool_pre_ping=True,   # 取连接前先 ping，自动丢弃死连接
    pool_recycle=1800,    # 连接最多用 30 分钟就回收
)
```
