# 牛马日记 — 服务器部署方案

## 目标服务器

http://39.106.63.179:8089/（与金毛大战波斯猫等项目共用）

## 项目占用资源

| 资源 | 位置 / 说明 | 预估占用 |
|------|------------|---------|
| 项目目录 | `/home/work/ai_hustle_diary/server/` | ~50 MB |
| MySQL | 复用线上已有的 1Panel-mysql-kaRl（mysql:8.4.8），库名 `ai_golden_vs_persian`，表统一用 `hd_` 前缀 | <10 MB |
| uv + Python 依赖 | `/root/.local/bin/uv` + `.venv/` | ~200 MB |
| systemd 服务 | `/etc/systemd/system/ai_hustle_diary.service` | 1 个文件 |
| **合计新增** | | **~250 MB**（不再部署独立数据库） |

## 服务器需要修改的地方

### 1. Nginx 配置（已有文件，改一处）

文件：`/etc/nginx/sites-available/game.yunque.co`

```diff
  location /hustle/ {
-     proxy_pass http://127.0.0.1:8066;
+     proxy_pass http://127.0.0.1:8089/;
  }
```

> 多个小游戏共用 `game.yunque.co` 域名，通过路径前缀区分：  
> - `/hustle/*` → 牛马日记（:8089）  
> - 其他游戏保持原有 location 不变

### 2. 初始化 MySQL 表（不新增容器）

直接在现有 MySQL 的 `ai_golden_vs_persian` 库里创建本项目的 2 张表：

```bash
docker exec -i 1Panel-mysql-kaRl \
    mysql -uroot -p$MYSQL_ROOT_PASSWORD ai_golden_vs_persian \
    < /home/work/ai_hustle_diary/server/sql/schema.sql
```

> 线上 MySQL root 密码见 `server/.env` 的 `MYSQL_ROOT_PASSWORD`。应用 `.env` 中 `MYSQL_HOST=39.106.63.179`、`MYSQL_DATABASE=ai_golden_vs_persian`。

### 3. 新增 systemd 服务

```bash
# /etc/systemd/system/ai_hustle_diary.service
ExecStart=uv run uvicorn main:app --host 0.0.0.0 --port 8089
WorkingDirectory=/home/work/ai_hustle_diary/server
```

### 4. 安装 uv（如未安装）

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

## 端口分配

| 端口 | 用途 |
|------|------|
| 8089 | 牛马日记后端（FastAPI） |
| 3306 | 共用 MySQL（容器 1Panel-mysql-kaRl）|

## 前端配置

```js
// client/global.js
GameGlobal.serverUrl = 'https://game.yunque.co/hustle'
```

前端请求 `serverUrl + '/api/auth/login'` → Nginx 去掉 `/hustle` 前缀 → 后端收到 `/api/auth/login`，后端代码无需修改。
