# 数据库总览

## 技术选型

- **主数据库**：MySQL 8.x — 与同服项目「金毛大战波斯猫」共用同一个 MySQL 实例与库
  - 本地：`golden_vs_persian`
  - 线上：`ai_golden_vs_persian`
  - 本项目所有表统一使用 `hd_` 前缀（hd = hustle diary / 牛马日记），与金毛项目的 `users` / `levels` / `rankings` 等表共存不冲突
- **ORM / 驱动**：SQLAlchemy 2.0 + asyncmy（异步 MySQL）

## 数据表设计（仅 2 张，最小功能集）

建表脚本见 [server/sql/schema.sql](../../server/sql/schema.sql)。

### 1. 用户表 `hd_users`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT, PK, AUTO_INCREMENT | 用户ID |
| openid | VARCHAR(64), UNIQUE | 微信 OpenID |
| nickname | VARCHAR(128) | 昵称 |
| avatar_url | VARCHAR(512) | 头像地址 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间（ON UPDATE） |

### 2. 通关记录表 `hd_records`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT, PK, AUTO_INCREMENT | 记录ID |
| user_id | BIGINT, FK → hd_users.id | 用户ID |
| level_id | INT | 关卡ID |
| clear_time | DOUBLE | 通关用时（秒） |
| created_at | DATETIME | 记录时间 |

索引：`idx_user_id(user_id)`、`idx_level_id(level_id)`、`idx_level_time(level_id, clear_time)`（排行榜查询）。

## ER 关系

```
hd_users (1) ──→ (N) hd_records
```

## 初始化与迁移

- 本地第一次启动：`mysql -h <host> -u root -p < server/sql/schema.sql`
- 应用启动时也会通过 SQLAlchemy 的 `Base.metadata.create_all` 自动建表（开发便利），但正式环境以 `schema.sql` 为准
- 库名通过环境变量 `MYSQL_DATABASE` 切换（本地 `golden_vs_persian`、线上 `ai_golden_vs_persian`），无需改代码
