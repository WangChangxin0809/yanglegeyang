# 前端总览

## 技术选型

- **平台**：微信小游戏
- **渲染**：Canvas 2D API（纯原生，不依赖任何引擎）
- **语言**：JavaScript（CommonJS 模块化）
- **屏幕适配**：全局比例参数体系，运行时根据实际屏幕尺寸动态计算像素值

## 场景架构

所有场景继承 `Scene` 基类，通过 `switchScene()` 切换，单例复用。

| 场景 | 文件 | 职责 |
|------|------|------|
| 加载 | `loading.js` | 预加载图片 + 音效资源，显示进度条 |
| 授权 | `auth.js` | 微信授权按钮，完成 `wx.login` + `update-profile` 后进菜单 |
| 主菜单 | `menu.js` | 标题动效、开始按钮、排行榜入口 |
| 游戏 | `game.js` | 关卡生命周期、动画/特效队列、道具触发、复活 & 结算流程 |
| 排行榜 | `rank.js` | 关卡 Tab、滚动列表、前三奖牌、当前用户成绩 |

场景跳转链路：`Loading → (Auth) → Menu → Game ⇄ Menu ⇄ Rank`

## 游戏界面布局

垂直分为五层（去除暂存区后的简化版），所有位置参数基于屏幕比例：

| 区域 | 屏幕位置 | 说明 |
|------|----------|------|
| 标题区 | 0%–10% | 关卡标题 + 返回按钮 + 实时计时器 |
| 卡牌区 | 10%–70% | 多区域卡牌堆叠布局 |
| moveOut 展示区 | 70%–80% | 移出道具专属区域（最多 3 张，不再是"暂存区") |
| 槽位区 | 82%–90% | 7 格收集槽位 |
| 道具区 | 90%–100% | 4 个道具按钮（移出 / 撤回 / 洗牌 / 透视） |

## 核心玩法逻辑

1. **卡牌堆叠**：多层卡牌按关卡配置堆叠，上层遮挡下层
2. **点击消除**：点击未被遮挡的卡牌 → 飞行动画 → 移入槽位
3. **三消判定**：槽位中凑齐 3 张相同卡牌自动消除（带闪光 + 粒子特效）
4. **胜负判定**：全部卡牌消除 = 通关；槽位满 7 张 = 失败
5. **道具系统**
   - **移出**：将槽位头部卡牌挪到棋盘下方的 moveOut 展示区，可点击回归
   - **撤回**：撤销上一步，卡牌恢复（三消后历史失效即清空）
   - **洗牌**：随机重排棋盘上所有未移除卡牌
   - **透视**：3 秒内显示所有底层卡牌
6. **复活机制**：失败时弹出复活弹窗，选择分享/看广告继续挑战（仅触发 moveOut 不 undo）
7. **全屏结算特效**：胜利 / 失败统一以全屏动态特效呈现（endFx），替代传统弹窗
8. **关卡计时器**：`game.js` 维护 `this.levelStartTime`，`renders/timer.js` 每帧显示实时用时，通关后随分享文案一起提交

## 关卡配置结构

每关配置独立文件（`scenes/game/levels/levelN.js`），导出对象包含：

| 字段 | 层级 | 说明 |
|------|------|------|
| `title` | 关卡级 | 关卡标题 |
| `iconTypes` | 关卡级 | 图标种类数（从 18 种中随机挑选） |
| `regions[].x, y` | 区域级 | 区域左上角（相对卡牌可用区域 0–1） |
| `regions[].layers[].gapRatio` | 层级 | 间距占卡牌尺寸的比例 |
| `regions[].layers[].cards[]` | 卡牌级 | 卡牌网格位置 `{ col, row }` |

全局统一参数常量（按职责分散到各 `renders/*.js` 中，遵循 *UI 参数全局常量化规范*）：

- `renders/cards.js`：`CARD_SIZE_SCALE = 0.12`、`CARD_3D_DEPTH = 0.1`、`BOARD_TOP/BOTTOM/SIDE`
- `renders/slots.js`：`SLOT_TOP = 0.82` 等
- `renders/props.js`：`PROP_COUNT = 4`、`PROP_TOP = 0.90` 等
- `renders/timer.js`：`TIMER_FONT_SCALE`、`TIMER_TOP`
- `renders/endFx.js`：结算特效文字位置与持续时间常量

## 模块职责分离

| 模块 | 文件 | 职责 |
|------|------|------|
| 入口 | `game.js` | 画布初始化、场景切换、事件分发、主循环（`requestAnimationFrame` + `dt`） |
| 游戏场景 | `scenes/game.js` | 生命周期、状态机、动画/特效队列、道具分发、复活 & 结算流程 |
| 渲染 | `scenes/game/renders/*` | 按职责拆分的独立渲染模块，遵循 `render(ctx, config, ...)` 统一签名 |
| 弹窗 | `scenes/game/dialogs/*` | 模块化弹窗（confirm / revive / win / propShare）+ 统一分发 |
| 游戏逻辑 | `scenes/game/gameLogic.js` | 卡牌生成、遮挡判定 |
| 关卡数据 | `scenes/game/levels.js` + `levels/` | 关卡配置汇总 |
| 道具系统 | `scenes/game/props/*` | 4 个道具独立实现（含分享解锁次数） |

## 屏幕适配方案

- 所有 UI 位置/尺寸参数均为屏幕宽度或高度的比例值
- 运行时通过 `wx.getWindowInfo()` 获取实际屏幕尺寸动态计算
- `game.json` 配置 `resizable: true`、`displayMode: "raw"` 确保原始分辨率
- 高清屏适配：canvas 缓冲按 `screenWidth * DPR` 分配，`ctx` 统一用逻辑像素绘制

## 与后端交互

- 通过 `utils/request.js` 封装的 `get()` / `post()` / `login()` / `authorize()` 与后端通信
- **登录**：`wx.login()` → `POST /api/auth/login` 换 JWT token；持久化到 `wx.setStorageSync('hd_token')`
- **授权**：loading 完成后无 token → 强制进入 `scenes/auth.js`，点击微信按钮触发 `authorize()`（= `login` + `update-profile`）
- **通关上报**：`POST /api/record/submit` 提交关卡 ID + 通关时间（秒）
- **排行榜**：`GET /api/record/rank?level_id=N&limit=50`（无需登录）
- 后端地址配置在 `global.js` 的 `GameGlobal.serverUrl`
