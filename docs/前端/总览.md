# 前端总览

## 技术选型

- **平台**：微信小游戏
- **渲染**：Canvas 2D API（纯 API 模式）
- **语言**：JavaScript（CommonJS 模块化）
- **屏幕适配**：全局百分比参数体系，运行时根据屏幕尺寸动态计算像素值

## 场景架构

所有场景继承 `Scene` 基类，通过 `switchScene()` 切换，单例复用。

| 场景 | 文件 | 职责 |
|------|------|------|
| 加载 | `loading.js` | 预加载菜单/游戏资源，显示进度条 |
| 主菜单 | `menu.js` | 标题晃动动效、开始按钮缩放动效、排行榜入口、动物装饰 |
| 游戏 | `game.js` | 卡牌交互、飞行动画、消除特效、粒子系统、透视模式、暂存区、撤回历史、关卡切换 |
| 排行榜 | `rank.js` | 关卡 Tab 切换、触摸滚动列表、前三名奖牌、当前用户排名 |

场景跳转链路：`Loading → Menu → Game ⇄ Menu ⇄ Rank`

## 游戏界面布局

垂直分为五层，所有位置参数基于屏幕比例：

| 区域 | 屏幕位置 | 说明 |
|------|----------|------|
| 标题区 | 0%–10% | 关卡标题 + 返回按钮 |
| 卡牌区 | 10%–65% | 多区域卡牌堆叠布局 |
| 暂存区 | 76%–82% | 移出道具暂存的卡牌（最多 3 张，点击可回归槽位） |
| 槽位区 | 82%–90% | 7 格收集槽位 |
| 道具区 | 90%–100% | 4 个道具按钮（移出/撤回/洗牌/透视） |

## 核心玩法逻辑

1. **卡牌堆叠**：多层卡牌按关卡配置堆叠，上层遮挡下层
2. **点击消除**：点击未被遮挡的卡牌 → 飞行动画 → 移入槽位
3. **三消判定**：槽位中凑齐 3 张相同卡牌自动消除（带特效 + 粒子）
4. **胜负判定**：全部卡牌消除且暂存区为空 = 通关；槽位满 7 张 = 失败
5. **道具系统**：
   - **移出**：将槽位头部卡牌移到暂存区（最多 3 张），点击暂存卡牌可回归槽位
   - **撤回**：撤销上一步操作，卡牌恢复到棋盘或暂存区原位（三消后历史清空）
   - **洗牌**：随机重新分配棋盘上所有未移除卡牌的图标，保持数量不变
   - **透视**：进入透视选择模式，点击顶层卡牌后周围 3×3 区域顶层卡牌变透明 3 秒，显示底层卡牌
6. **退出确认**：点击返回按钮弹出确认对话框，防止误操作

## 关卡配置结构

每关配置独立文件（`levels/levelN.js`），导出对象包含：

| 字段 | 层级 | 说明 |
|-------|------|------|
| `title` | 关卡级 | 关卡标题 |
| `iconTypes` | 关卡级 | 图标种类数（从 18 种中随机挑选） |
| `regions[].x, y` | 区域级 | 区域左上角（相对可用区域 0–1） |
| `regions[].layers[].gapRatio` | 层级 | 间距占卡牌尺寸的比例 |
| `regions[].layers[].cards[]` | 卡牌级 | 卡牌网格位置 `{ col, row }` |

全局统一参数（定义在 `cardRender.js`）：
- 卡牌尺寸 `CARD_SIZE_SCALE = 0.12`
- 3D 厚度 `CARD_3D_DEPTH = 0.1`
- 暂存区位置 `STASH_TOP = 0.76`
- 道具数量 `PROP_COUNT = 4`

## 模块职责分离

| 模块 | 文件 | 职责 |
|------|------|------|
| 场景管理 | `game.js`（入口） | 画布初始化、场景切换、事件分发、主循环 |
| 游戏场景 | `scenes/game.js` | 生命周期、背景/HUD 渲染、动画队列、透视模式、暂存区交互、撤回历史、关卡切换 |
| 卡牌渲染 | `game/cardRender.js` | 图片预加载、卡牌生成、遮挡判定、绘制卡牌/槽位/暂存区/道具区、透视渲染 |
| 游戏逻辑 | `game/gameLogic.js` | 点击检测、槽位管理、三消匹配、输赢判定（含暂存区状态） |
| 关卡数据 | `game/levels.js` + `levels/` | 关卡配置汇总与独立文件 |
| 道具系统 | `game/props/` | 移出、撤回、洗牌、透视（独立文件 + 统一入口） |

## 屏幕适配方案

- 所有 UI 位置/尺寸参数均为屏幕宽度或高度的比例值
- 运行时通过 `wx.getWindowInfo()` 获取实际屏幕尺寸动态计算
- `game.json` 配置 `resizable: true`、`displayMode: "raw"` 确保原始分辨率
- 卡牌可用区域由 `BOARD_TOP`、`BOARD_BOTTOM`、`BOARD_SIDE` 定义，区域内坐标为 0–1 比例

## 与后端交互

- 通过 `utils/request.js` 封装的 `get()` / `post()` 与后端 API 通信
- **登录**：`wx.login()` 获取 code → `POST /api/user/login` 换取 token（本地开发走 `/api/user/dev-login` 免 code）
- **通关上报**：`POST /api/record/submit` 提交关卡 ID + 通关时间
- **排行榜**：`GET /api/record/rank?level_id=N&limit=50` 获取指定关卡排名（无需登录）
- 后端地址配置在 `global.js` 的 `GameGlobal.serverUrl`
