# 游戏界面比例调整指南

所有界面参数均为**屏幕比例值**（0~1），运行时根据设备实际屏幕尺寸动态计算像素值。修改对应常量即可调整布局，无需改动任何逻辑代码。

所有 UI 参数遵循 *UI 参数全局常量化规范* —— **不使用魔法数字**，每个 render 模块在文件顶部以 `xxx_SCALE / xxx_TOP / xxx_COLOR` 的形式集中声明。

## 界面纵向布局总览

```
  0%  ┌──────────────────────┐
      │     标题区 + 计时器   │  TITLE_TOP / TIMER_TOP
 10%  ├──────────────────────┤
      │                      │
      │     卡牌区            │  BOARD_TOP ~ BOARD_BOTTOM
      │                      │
 70%  ├──────────────────────┤
      │   moveOut 展示区      │  MOVEOUT_Y
 80%  ├──────────────────────┤
      │     槽位区            │  SLOT_TOP
 90%  ├──────────────────────┤
      │     道具区            │  PROP_TOP
100%  └──────────────────────┘
```

---

## 参数所在文件一览

所有可调参数分散在 `client/scenes/game/renders/*.js` 各自模块里，修改时按职责定位：

| 关心区域 | 文件 |
|---------|------|
| 卡牌尺寸 / 可用区域 / 遮挡阈值 | `renders/cards.js` |
| 槽位 | `renders/slots.js` |
| moveOut 展示区 | `renders/cards.js`（`MOVEOUT_Y` 等常量） |
| 道具按钮 | `renders/props.js` |
| 标题文字 | `renders/title.js` |
| 关卡计时器 | `renders/timer.js` |
| 返回按钮 | `renders/backButton.js` |
| 全屏结算特效 | `renders/endFx.js` |
| Toast 提示 | `renders/toast.js` |

## 参数说明

### 1. 卡牌可用区域（renders/cards.js）

控制卡牌能放置的矩形区域边界。

| 参数 | 当前值 | 基准 | 说明 |
|------|--------|------|------|
| `BOARD_TOP` | 0.10 | 屏幕高度 | 卡牌区顶部，紧贴标题下方 |
| `BOARD_BOTTOM` | 0.70 | 屏幕高度 | 卡牌区底部 |
| `BOARD_SIDE` | 0.07 | 屏幕宽度 | 卡牌区左右边距 |

**调整示例：**
- 想让卡牌区更高 → 增大 `BOARD_BOTTOM`（如 0.72）
- 想让卡牌区左右更窄 → 减小 `BOARD_SIDE`（如 0.04）
- 想让卡牌区从更高开始 → 减小 `BOARD_TOP`（如 0.08）

### 2. 卡牌绘制参数（renders/cards.js）

控制单张卡牌的外观。

| 参数 | 当前值 | 基准 | 说明 |
|------|--------|------|------|
| `CARD_SIZE_SCALE` | 0.12 | 屏幕宽度 | 卡牌尺寸（宽=高），所有关卡统一 |
| `CARD_3D_DEPTH` | 0.1 | 卡牌尺寸 | 3D 侧面厚度 |
| `CARD_RADIUS` | 8 | px | 卡牌圆角半径 |
| `CARD_ICON_PAD` | 0.07 | 卡牌尺寸 | 图标距卡牌边缘的内边距 |
| `CARD_FONT_SCALE` | 0.45 | 卡牌尺寸 | 卡牌文字字号（无图片时的兜底显示） |
| `CARD_BLOCKED_THR` | 0.1 | 卡牌面积 | 遮挡判定阈值，重叠面积超过此比例则视为被遮挡 |
| `CARD_BLOCKED_COLOR` | `#a0a5a8` | — | 被遮挡卡牌的灰化背景色（*isblocked 卡牌背景色规范*） |

### 3. moveOut 展示区参数（renders/cards.js）

移出道具使用后，卡牌会出现在棋盘下方的专属展示区（**已不再使用"暂存区"概念**）。

| 参数 | 当前值 | 基准 | 说明 |
|------|--------|------|------|
| `MOVEOUT_Y` | 0.72 | 屏幕高度 | moveOut 展示区顶部位置（像素语义 / 非比例已在坐标规则里澄清） |
| `MOVEOUT_MAX` | 3 | — | 最多可同时展示的移出卡牌数 |

### 4. 槽位区参数（renders/slots.js）

控制底部 7 格收集槽位的位置和外观。

| 参数 | 当前值 | 基准 | 说明 |
|------|--------|------|------|
| `SLOT_TOP` | 0.82 | 屏幕高度 | 槽位区顶部位置 |
| `SLOT_SIZE_SCALE` | 0.11 | 屏幕宽度 | 单个槽位的尺寸 |
| `SLOT_GAP_SCALE` | 0.015 | 屏幕宽度 | 槽位之间的间距 |
| `SLOT_PAD_SCALE` | 0.02 | 屏幕宽度 | 槽位背景的内边距 |
| `SLOT_ICON_PAD` | 0.06 | 槽位尺寸 | 槽位内图标的内边距 |

### 5. 道具区参数（renders/props.js）

控制底部 **4 个**道具按钮（`PROP_COUNT = 4`，分别对应 moveOut / undo / shuffle / peek）的位置和外观。

| 参数 | 当前值 | 基准 | 说明 |
|------|--------|------|------|
| `PROP_TOP` | 0.90 | 屏幕高度 | 道具区顶部位置 |
| `PROP_SIZE_SCALE` | 0.15 | 屏幕宽度 | 道具按钮尺寸（正方形） |
| `PROP_GAP_SCALE` | 0.04 | 屏幕宽度 | 道具按钮之间的间距 |
| `PROP_COUNT` | 4 | — | 道具按钮数量 |
| `PROP_RADIUS` | 10 | px | 道具按钮圆角半径 |
| `PROP_FONT_SCALE` | 0.28 | 按钮尺寸 | 道具按钮剩余次数字号 |

> 注意：4 个按钮的总宽度 ≈ `4 × PROP_SIZE_SCALE + 3 × PROP_GAP_SCALE`，不要超出屏幕宽度。

### 6. 标题区参数（renders/title.js）

| 参数 | 当前值 | 基准 | 说明 |
|------|--------|------|------|
| `TITLE_FONT_SCALE` | 0.055 | 屏幕宽度 | 关卡标题字号 |
| `TITLE_TOP` | 0.02 | 屏幕高度 | 标题顶部位置 |

### 7. 关卡计时器参数（renders/timer.js）

| 参数 | 当前值 | 基准 | 说明 |
|------|--------|------|------|
| `TIMER_FONT_SCALE` | 0.04 | 屏幕宽度 | 计时器字号 |
| `TIMER_TOP` | 0.11 | 屏幕高度 | 计时器顶部位置 |
| `TIMER_COLOR` | `#0a0a0a` | — | 计时器文字颜色 |

### 8. 返回按钮参数（renders/backButton.js）

| 参数 | 当前值 | 基准 | 说明 |
|------|--------|------|------|
| `BACK_SIZE_SCALE` | 0.09 | 屏幕宽度 | 按钮直径 |
| `BACK_X_SCALE` | 0.03 | 屏幕宽度 | 按钮左边距 |
| `BACK_Y_SCALE` | 0.015 | 屏幕高度 | 按钮顶边距 |

### 9. 全屏结算特效（renders/endFx.js）

胜利 / 失败时会**全屏覆盖**动态特效（`endFx`），取代原来的结算弹窗（见 *游戏结算反馈采用全屏覆盖特效* 决策）。

| 参数 | 当前值 | 基准 | 说明 |
|------|--------|------|------|
| `END_FX_DURATION` | 4000 | ms | 特效持续时间 |
| `END_FX_TITLE_Y` | 0.40 | 屏幕高度 | 大标题纵向位置 |
| `END_FX_SUB_OFFSET` | 1.5 | 标题字号倍数 | 副标题相对标题的偏移 |
| `END_FX_MASK_ALPHA` | 0.55 | — | 失败深灰蒙层透明度（覆盖在 lose.png 之上） |

## 调整注意事项

1. **区域不要重叠**：确保 `BOARD_BOTTOM` < `MOVEOUT_Y` < `SLOT_TOP` < `PROP_TOP`，各区域之间留出间距
2. **槽位总宽度**：`7 × SLOT_SIZE_SCALE + 6 × SLOT_GAP_SCALE` 不要超过 `1 - 2 × SLOT_PAD_SCALE`
3. **道具区总宽度**：`4 × PROP_SIZE_SCALE + 3 × PROP_GAP_SCALE ≤ 0.95`
4. **卡牌区容量**：增大 `CARD_SIZE_SCALE` 后关卡配置中的卡牌网格可能放不下，需同步调整关卡布局
5. **修改后清缓存**：在微信开发者工具中修改参数后，建议清除全部缓存重新编译
6. **统一在 renders/ 下修改**：切勿把参数回流到 `scenes/game.js` 里形成魔法数字，违反项目 *UI 参数全局常量化规范*
