# 音频资源需求

> 本文档面向**音频资源提供者**和**上传者**，说明音频文件的存放规范、命名规则以及上传到阿里云 OSS 的操作流程。

## 1. 存放目录（当前实际结构）

所有音频源文件统一放到 `client/audio/`，按用途分三类：

```
client/audio/
├── bg_home.mp3              # 主页（菜单）循环背景乐
├── bg_game/                 # 游戏内循环背景乐（多首随机）
│   ├── bg_game.mp3
│   ├── bg_game_2.mp3
│   └── bg_game_3.mp3
└── game/                    # 游戏内触发式音效
    ├── click.mp3            #   通用点击 / 卡牌点击
    ├── merge.mp3            #   三消成功
    ├── success.mp3          #   通关胜利
    └── defeat.mp3           #   槽位爆满失败
```

> 注：历史版本的 `bgm/` + `sfx/` 分类已废弃，改为「主页 BGM 单文件 + 游戏 BGM 目录 + 游戏 SFX 目录」的三段式结构，见 [client/utils/audio.js](../../client/utils/audio.js) 的封装。

## 2. 格式与规范

| 项目 | 要求 | 说明 |
|------|------|------|
| 格式 | `.mp3`（首选）或 `.m4a` | 微信小游戏 `InnerAudioContext` 全平台兼容 |
| 采样率 | 44.1 kHz | |
| 声道 | 立体声（BGM）/ 单声道（SFX）| 音效单声道足够，减小体积 |
| 码率 | BGM 128 kbps / SFX 96 kbps | |
| 单文件大小 | BGM ≤ 1 MB，SFX ≤ 50 KB | 控制小游戏总包体与加载时间 |
| 时长 | BGM 30–90 秒无缝循环；SFX ≤ 1.5 秒 | |
| 命名 | 全小写 + 下划线 | `click.mp3` 而非 `Click.MP3` |
| 版权 | 必须可商用或自制 | 禁止直接搬运他人素材 |

## 3. 需求清单

| 文件路径 | 类型 | 时长 | 说明 |
|---------|------|------|------|
| `audio/bg_home.mp3` | BGM | 60 s | 主菜单 / 授权 / 排行场景循环背景乐，轻松活泼 |
| `audio/bg_game/bg_game.mp3` | BGM | 60 s | 游戏内循环 BGM（默认） |
| `audio/bg_game/bg_game_2.mp3` | BGM | 60 s | 游戏内循环 BGM（随机切换） |
| `audio/bg_game/bg_game_3.mp3` | BGM | 60 s | 游戏内循环 BGM（随机切换） |
| `audio/game/click.mp3` | SFX | ≤ 0.5 s | 按钮 / 卡牌点击 |
| `audio/game/merge.mp3` | SFX | ≤ 1.0 s | 三消成功特效 |
| `audio/game/success.mp3` | SFX | ≤ 1.5 s | 通关胜利 |
| `audio/game/defeat.mp3` | SFX | ≤ 1.5 s | 槽位爆满失败 |

> 当前代码未单独为「道具触发」配音效，点击道具按钮复用 `click.mp3`；如未来需要独立音效，请在 `audio/game/` 下补充 `prop.mp3` 并在 [client/utils/audio.js](../../client/utils/audio.js) 注册。

## 4. OSS 路径规范

音频对象键使用**独立前缀** `v1/audio/`，与图片 `v1/images/` 并列：

```
OSS 对象键                             ← 本地相对路径
v1/audio/bg_home.mp3                    ← client/audio/bg_home.mp3
v1/audio/bg_game/bg_game.mp3            ← client/audio/bg_game/bg_game.mp3
v1/audio/game/click.mp3                 ← client/audio/game/click.mp3
```

公网访问 URL（示例）：

```
https://yanglegeyang-assets.oss-cn-shanghai.aliyuncs.com/v1/audio/bg_home.mp3
```

> Bucket、Endpoint、AccessKey 与图片上传复用 [server/.env](../../server/.env) 的 `OSS_*` 变量。

## 5. 上传方式

### 方式 A：脚本批量同步（已落地，推荐）

使用项目内置脚本 [scripts/oss_upload/upload_audio.py](../../scripts/oss_upload/upload_audio.py) 一键同步：

```powershell
cd scripts/oss_upload
uv run python upload_audio.py --dry-run     # 预览将要上传的文件
uv run python upload_audio.py               # 正式上传
```

脚本会：
- 扫描 `client/audio/` 下所有 `.mp3` / `.m4a` 文件
- 按相对路径映射到 OSS `v1/audio/` 前缀
- 增量上传（跳过已存在且哈希一致的文件）
- 生成清单到 `client/oss_manifest.json`

### 方式 B：OSS 控制台手动上传（临时补传）

1. 登录阿里云 OSS 控制台，进入 bucket `yanglegeyang-assets`。
2. 切到目录 `v1/audio/`（不存在就新建），再按 `bg_game/` 与 `game/` 建子目录。
3. 上传文件时**权限读写**选择"公共读"（或继承 bucket 策略）。
4. 上传完成后在浏览器访问 URL 验证能直接播放。

## 6. 前端接入

音频资源通过 [client/utils/audio.js](../../client/utils/audio.js) 统一管理，外部调用只需语义方法：

```js
const audio = require('./utils/audio')

audio.playHomeBgm()           // 进入主页 / 菜单自动调用
audio.playGameBgm()           // 进入 game 场景时随机一首 bg_game/*
audio.stopBgm()               // 切场景前停掉背景乐

audio.playClick()             // 按钮 / 卡牌点击
audio.playMerge()             // 三消成功（gameLogic.js 内）
audio.playSuccess()           // 通关（win 弹窗展示时）
audio.playDefeat()            // 失败（lose 结算特效时）
```

远程 / 本地切换由 [client/utils/assets.js](../../client/utils/assets.js) 的 `USE_REMOTE_ASSETS` 开关统一控制，无需在业务代码中写完整 URL。

> **注意**：远程 URL 需在微信公众平台的 `downloadFile` 合法域名白名单中添加 `https://yanglegeyang-assets.oss-cn-shanghai.aliyuncs.com`；开发者工具可先勾选"不校验域名"调试。

## 7. 上线前检查清单

- [ ] 所有音频文件放在 `client/audio/` 对应子目录（`bg_home.mp3` / `bg_game/` / `game/`）
- [ ] 命名全小写、后缀统一 `.mp3`
- [ ] 单文件大小符合第 2 节限制
- [ ] OSS 上已存在并可公网播放
- [ ] 微信公众平台 downloadFile 域名白名单已添加
- [ ] 代码中通过 [client/utils/audio.js](../../client/utils/audio.js) 的语义方法调用，未硬编码完整 URL
