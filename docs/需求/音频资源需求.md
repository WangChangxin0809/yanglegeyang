# 音频资源需求

> 本文档面向**音频资源提供者**和**上传者**，说明音频文件的存放规范、命名规则以及上传到阿里云 OSS 的操作流程。

## 1. 存放目录

所有音频源文件统一放到 `client/audio/`，按用途分子目录：

```
client/audio/
├── bgm/                     # 背景音乐（循环播放）
│   ├── menu_bgm.mp3         #   主菜单背景乐
│   └── game_bgm.mp3         #   游戏场景背景乐
└── sfx/                     # 音效（触发式）
    ├── click.mp3            #   通用点击
    ├── card_pick.mp3        #   选中卡牌
    ├── match.mp3            #   三消成功
    ├── win.mp3              #   通关胜利
    ├── lose.mp3             #   失败
    └── prop.mp3             #   道具使用
```

## 2. 格式与规范

| 项目 | 要求 | 说明 |
|------|------|------|
| 格式 | `.mp3`（首选）或 `.m4a` | 微信小游戏 `InnerAudioContext` 全平台兼容 |
| 采样率 | 44.1 kHz | |
| 声道 | 立体声（BGM）/ 单声道（SFX）| 音效单声道足够，减小体积 |
| 码率 | BGM 128 kbps / SFX 96 kbps | |
| 单文件大小 | BGM ≤ 1 MB，SFX ≤ 50 KB | 控制小游戏总包体与加载时间 |
| 时长 | BGM 30–90 秒无缝循环；SFX ≤ 1.5 秒 | |
| 命名 | 全小写 + 下划线，见下表 | `click.mp3` 而非 `Click.MP3` |
| 版权 | 必须可商用或自制 | 禁止直接搬运他人素材 |

## 3. 需求清单

| 文件路径 | 类型 | 时长 | 说明 |
|---------|------|------|------|
| `audio/bgm/menu_bgm.mp3` | BGM | 60 s | 主菜单循环背景乐，轻松活泼 |
| `audio/bgm/game_bgm.mp3` | BGM | 60 s | 游戏中循环背景乐，节奏略紧凑 |
| `audio/sfx/click.mp3` | SFX | ≤ 0.3 s | 按钮点击 |
| `audio/sfx/card_pick.mp3` | SFX | ≤ 0.5 s | 卡牌被点击飞向槽位 |
| `audio/sfx/match.mp3` | SFX | ≤ 1.0 s | 三消成功特效 |
| `audio/sfx/win.mp3` | SFX | ≤ 1.5 s | 通关 |
| `audio/sfx/lose.mp3` | SFX | ≤ 1.5 s | 失败 |
| `audio/sfx/prop.mp3` | SFX | ≤ 0.5 s | 道具触发（洗牌/撤回/透视/移出通用） |

## 4. OSS 路径规范

音频对象键使用**独立前缀** `v1/audio/`，与图片 `v1/images/` 并列：

```
OSS 对象键        ← 本地相对路径
v1/audio/bgm/menu_bgm.mp3   ← client/audio/bgm/menu_bgm.mp3
v1/audio/sfx/click.mp3      ← client/audio/sfx/click.mp3
```

公网访问 URL：

```
https://yanglegeyang-assets.oss-cn-shanghai.aliyuncs.com/v1/audio/bgm/menu_bgm.mp3
```

> Bucket、Endpoint、AccessKey 与图片上传复用 [server/.env](../../server/.env) 的 `OSS_*` 变量。

## 5. 上传方式

### 方式 A：OSS 控制台手动上传（推荐临时使用）

1. 登录阿里云 OSS 控制台，进入 bucket `yanglegeyang-assets`。
2. 切到目录 `v1/audio/`（不存在就新建），再按 `bgm/` 与 `sfx/` 建子目录。
3. 上传文件时**权限读写**选择"公共读"（或继承 bucket 策略）。
4. 上传完成后在浏览器访问 URL 验证能直接播放。

### 方式 B：用上传脚本批量同步（需先扩展脚本）

当前 [upload.py](../../scripts/oss_upload/upload.py) 只处理 `client/images/` 且前缀写死 `v1/images`，**不支持音频**。扩展要点：

1. `ALLOWED_EXT` 增加 `".mp3", ".m4a", ".wav", ".ogg"`。
2. `OSS_PREFIX` 与 `LOCAL_ROOT` 改为按资源类型可切换（新增 `--kind {images|audio}` 参数，或拆成 `upload_audio.py`）。
3. 清单输出路径独立，例如 `client/oss_audio_manifest.json`，避免覆盖图片清单。

扩展后用法（示例）：

```powershell
cd scripts/oss_upload
uv run python upload.py --kind audio --dry-run   # 预览
uv run python upload.py --kind audio             # 正式上传
```

## 6. 前端接入（开发者参考）

音频资源与图片对称，建议在 [client/utils/assets.js](../../client/utils/assets.js) 增加 `getAudioUrl`：

```js
const REMOTE_AUDIO_BASE = 'https://yanglegeyang-assets.oss-cn-shanghai.aliyuncs.com/v1/audio'
const LOCAL_AUDIO_BASE = 'audio'

function getAudioUrl(relPath) {
  const base = USE_REMOTE_ASSETS ? REMOTE_AUDIO_BASE : LOCAL_AUDIO_BASE
  return base + '/' + relPath
}
module.exports = { getImageUrl, getAudioUrl, USE_REMOTE_ASSETS }
```

播放示例：

```js
const { getAudioUrl } = require('./utils/assets')

const bgm = wx.createInnerAudioContext()
bgm.src = getAudioUrl('bgm/menu_bgm.mp3')
bgm.loop = true
bgm.play()

const sfx = wx.createInnerAudioContext()
sfx.src = getAudioUrl('sfx/match.mp3')
sfx.play()
```

> **注意**：远程 URL 需在微信公众平台的 `downloadFile` 合法域名白名单中添加 `https://yanglegeyang-assets.oss-cn-shanghai.aliyuncs.com`；开发者工具可先勾选"不校验域名"调试。
